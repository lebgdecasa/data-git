Run the owner's Monday Hour One. The Execution Coach 🏁 leads; keep theatrics to one sentence.

## Steps

1. **Load state:** read the most recent file in `foundry/ops/weeks/` (if any), `foundry/war-room/decisions.md` (active decisions = priority candidates), `foundry/ops/pipeline.md` (live prospects = scheduled follow-ups), and `foundry/vault/personal/profile.md` (hours/week available).
2. **The Look Back** (skip on first run): walk last week's plan — Done / Not done / Why. Identify last week's #1 constraint and propose ONE process/checklist that prevents it recurring (offer to save it under `foundry/ops/`).
3. **The Look Forward:** from active decisions, pipeline next-actions, and anything the owner adds in the arguments — list candidate tasks for the coming 1–3 months and pull forward what must happen THIS week.
4. **Prioritize:** force-rank. The top 1–3 are PRIORITIES — everything else schedules around them, never the reverse. Per doctrine, business-building blocks (outreach, offer work, delivery) outrank admin.
5. **Schedule:** propose the week as named time blocks (no generic "project work" — "Send 20 cold DMs to French dental clinics"), realistic (breaks, lunch, day-job hours if any), with explicit OVERFLOW/ad-hoc buffers. Ask the owner only for genuinely unknown fixed commitments — otherwise propose and let them correct.
6. **Color code** (for transfer to their real calendar): recurring meetings blue · admin red · one-off meetings yellow · priorities/projects green · breaks purple · personal orange.
7. **Save** to `foundry/ops/weeks/<YYYY>-W<nn>.md`: priorities, the block schedule (Mon–Sun table), success criteria for the week ("week is a WIN if..."), carried-over items, constraint-fix adopted.
8. Close with the Rule-of-100 check: does this week contain ≥100 primary outreach/content actions toward the active goal? If not, say so plainly and show what to cut to make room.
